Regularisation and Model Complexity

Regularisation adds a penalty for complexity to the model's objective function — shrinking coefficients toward zero, eliminating irrelevant features, and preventing overfitting. It is the primary tool for navigating the bias-variance tradeoff in linear and generalized linear models.

Terms covered: 14 Sections: Ridge · Lasso · ElasticNet · Hyperparameter tuning · Sparse models · Dropout
Section 12.1

Why regularisation works — penalising complexity

Standard OLS finds coefficients that minimize the sum of squared residuals on the training data. With many features, this produces overfitted models. Regularised regression adds a penalty term to the objective function — making the model pay a cost for large coefficients, which forces it to only use features that provide enough predictive value to outweigh their complexity cost.

Ridge Regression (L2 Regularisation)
Adds a penalty proportional to the sum of squared coefficients — shrinking all coefficients toward zero but retaining all features.
expand
Intuition
It tells you the model is prevented from placing extreme weights on any single feature — all coefficients are pulled toward zero in proportion to their size. Ridge is the appropriate choice when you believe all features are potentially relevant but want to prevent any single predictor from dominating. It is particularly effective at addressing multicollinearity — it distributes weight across correlated features rather than arbitrarily picking one.
Formula / Rule
Minimize: Σ(yᵢ−ŷᵢ)² + λΣβⱼ²λ = regularisation strength. λ=0: standard OLS. λ→∞: all coefficients shrink to zero. Optimal λ is chosen by cross-validation. Ridge shrinks all coefficients but never sets them exactly to zero — no automatic feature selection. Scale features before applying ridge (or any regularised regression).
When you see it
In marketing mix models with correlated media channels, genomics with many correlated genetic markers, economic forecasting with correlated macro variables, and any regression with multicollinearity.
Red flag
Ask whether features were standardised before applying Ridge. Ridge shrinkage is proportional to coefficient size — if features are on different scales, unstandardised Ridge penalises features with large natural coefficients more heavily, creating scale-dependent bias.
Business example
A media company's marketing mix model has TV, digital, social, and OOH spend as predictors, all highly correlated (r > 0.7 between channels). OLS produces unstable, counterintuitive negative coefficients for some channels. Ridge with λ selected by 10-fold CV produces stable positive elasticities for all channels — still retaining all, but distributing attribution sensibly across correlated inputs.
Lasso Regression (L1 Regularisation)
Adds a penalty proportional to the sum of absolute coefficient values — shrinking some to exactly zero, performing automatic feature selection.
expand
Intuition
It tells you the model is simultaneously selecting features and estimating their effects. The L1 penalty's geometry creates "corners" in the constraint region where exact zeros occur — Lasso naturally produces sparse models by eliminating irrelevant features entirely. This is ideal when you have many candidate features and believe most are genuinely irrelevant.
Formula / Rule
Minimize: Σ(yᵢ−ŷᵢ)² + λΣ|βⱼ|Higher λ → more coefficients set to zero → sparser model. λ optimal by CV (lambda.min gives best prediction; lambda.1se gives more parsimonious model within 1 SE). Lasso struggles when correlated predictors are both important — may arbitrarily pick one and zero the other. ElasticNet addresses this.
When you see it
In high-dimensional feature selection (500 candidate features, need to identify the 20 most predictive), gene expression analysis, text analytics with many sparse features, and any context where automated feature elimination is desired.
Red flag
Ask whether Lasso's sparsity is biologically/economically plausible in the domain. If the true model genuinely involves many small effects (common in social science and economics), Lasso's aggressive zeroing may create a sparser model than truth warrants, sacrificing predictive performance.
Business example
A telecom carrier's churn model starts with 340 candidate features. Lasso with λ selected by 5-fold CV retains 28 features (92% eliminated). The 28-feature Lasso model achieves AUC=0.857 vs 0.861 for the full 340-feature OLS — nearly identical predictive performance with 12× fewer features. The parsimonious model is easier to maintain, monitor, and explain to regulators.
ElasticNet
A combination of Ridge and Lasso penalties — getting automatic feature selection from L1 while handling correlated features better through L2.
expand
Intuition
It tells you the model is using both penalties simultaneously — L1 for sparsity and feature selection, L2 for stability among correlated features. When two features are highly correlated, Lasso arbitrarily picks one; ElasticNet tends to include both (if both are predictive) while still eliminating genuinely irrelevant features. It is the default regularisation choice for most high-dimensional regression problems.
Formula / Rule
Minimize: Σ(yᵢ−ŷᵢ)² + λ[α|β|₁ + (1−α)|β|₂²]Two hyperparameters: λ (overall penalty strength) and α (L1 vs L2 mixing ratio, 0=Ridge, 1=Lasso). Both selected by cross-validation. In scikit-learn: l1_ratio = α. Default starting point: α=0.5 or α=0.1 (light Lasso + heavy Ridge) with λ tuned by CV.
When you see it
As the default choice for regularised linear and logistic regression in most production ML pipelines, particularly in biological and financial applications with correlated feature groups.
Red flag
Ask whether both λ and α were tuned by cross-validation or whether one was fixed arbitrarily. Fixing α = 0.5 without validation may leave performance on the table if the data favours a very sparse (α near 1) or very smooth (α near 0) solution.
Business example
A pharmaceutical company predicts drug response from 1,000 biomarkers. Many biomarkers are highly correlated within pathways. Lasso selects one per pathway arbitrarily; Ridge retains all 1,000 with small coefficients. ElasticNet with α=0.3 identifies the 45 most predictive biomarkers (one to three per relevant pathway) with higher out-of-sample R² than either pure method — coherent biological interpretation and superior prediction simultaneously.
Hyperparameter Tuning
The process of selecting model configuration parameters (like λ) that are not learned from data but control how the model learns.
expand
Intuition
It tells you the regularisation strength, tree depth, learning rate, and number of estimators are not fitted to the training data — they are meta-parameters that control how the model fits. Setting them too aggressively overfits; too conservatively underfits. Cross-validation is used to empirically find values that minimise validation error.
Formula / Rule
Grid search: exhaustively try all combinations. Random search: sample hyperparameter space randomly — often more efficient for high-dimensional spaces. Bayesian optimisation: use past evaluations to guide next trials — most sample-efficient. Always use cross-validation within the training set — never touch the test set during tuning.
When you see it
In model development for any algorithm with hyperparameters — which is essentially all of them. The model selection pipeline is: define candidate hyperparameter ranges → CV-evaluate each configuration → select best configuration → evaluate once on test set.
Red flag
Ask whether the test set was used for hyperparameter selection. If performance on the test set was used to choose among hyperparameter configurations, the test set is no longer a valid measure of generalisation — a new holdout is needed.
Business example
A data team tunes a gradient boosting model using a grid of 80 hyperparameter combinations evaluated by 5-fold CV. The best configuration is selected by CV AUC, then evaluated once on the held-out test set. The test AUC (0.893) is within 0.008 of the best CV AUC (0.901) — a small gap suggesting the tuning process did not overfit the validation procedure itself.
Dropout
A regularisation technique for neural networks that randomly deactivates neurons during training — preventing co-adaptation and reducing overfitting.
expand
Intuition
It tells you the neural network cannot rely on any specific combination of neurons to jointly memorize training examples. By randomly setting a fraction of neurons to zero during each training pass, dropout forces the network to learn redundant representations — equivalent to training an ensemble of many thinned networks and averaging their predictions at inference time.
Formula / Rule
Dropout rate p: each neuron zeroed with probability p during trainingTypical rates: 0.2–0.5. At inference time: multiply all weights by (1−p) to maintain expected activation magnitude. Dropout is applied per-layer — early layers often use low dropout (0.1), deeper layers higher (0.3–0.5). Disable completely during evaluation and inference.
When you see it
In neural network architectures for any task — NLP, image recognition, tabular data, recommendation systems. Standard practice to include dropout in any deep network to reduce overfitting.
Red flag
Ask whether dropout was disabled during inference. Leaving dropout active at prediction time introduces randomness into predictions — the same input produces different outputs each call. This is sometimes used intentionally for uncertainty estimation (Monte Carlo dropout) but must be flagged if unintentional.
Business example
A fintech company trains a neural network for transaction categorisation. Without dropout, the 4-layer network achieves 94.3% training accuracy vs 81.2% validation — clear overfitting. Adding dropout (0.2 on layers 1–2, 0.4 on layers 3–4) closes the gap to 92.1% training vs 89.8% validation — the model now generalises substantially better to new transaction types.
Chapter Summary

Chapter 12 cheat sheet

Regularisation quick reference
Ridge (L2): correlated features
Shrinks all coefficients toward zero, retains all features. Best when features are correlated and all potentially relevant. Stabilizes multicollinear models.
Lasso (L1): automatic feature selection
Sets some coefficients exactly to zero. Best when most features are genuinely irrelevant. Produces sparse, interpretable models.
ElasticNet: the practical default
Combines both penalties. Handles correlated features better than Lasso while still providing sparsity. Default choice for most high-dimensional regressions.
Always standardise before regularising
Regularisation penalises coefficient magnitude — which is scale-dependent. Unstandardised features receive different penalty levels based on their units, not their importance.
Tune λ by cross-validation
Use the training and validation sets only. Never select λ based on test set performance — that contaminates your final evaluation.
Dropout for neural network regularisation
Rates 0.2–0.5. Must be disabled during inference. Equivalent to ensemble averaging over many thinned networks.
